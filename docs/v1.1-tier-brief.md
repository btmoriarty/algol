# Algol 0.10.0: the model_corroborated tier, build brief and diff

Draft for review, not yet applied to the repo. Decided 2026-07-23 after three product reviews: unqualified `verified` for a gauntlet `[V]` anchor is an overclaim for an honesty-first tool. This brief adds a `model_corroborated` tier below `verified` and enforces per-source ceilings so only a deterministic verifier can reach `verified`. (Queue name for the release is Algol v1.1; the repo version bumps to `0.10.0`. `1.0.0` stays reserved for after the managed check-run and public release.)

Validation run, 2026-07-23: this diff was applied to a throwaway copy of the repo. The 33 tier-relevant tests passed (record, gauntlet_adapter, reconcile, compose_adapter, and a new tier_ceilings suite). `test_fail_is_verified` stayed green, so uat's deterministic `verified` is intact; `test_never_upgrades_heuristic` now reads `model_corroborated`; the ceiling matrix confirms only uat reaches `verified`. The full suite was not run because the compile tests require a TOML backend; run `python3 -m unittest` from the repo root before committing.

## The rule

- `verified` means a deterministic verifier directly established the claimed condition and retained the evidence to reproduce it. It applies only to the condition the verifier actually checked. The reference verified source is `evidence-locked-uat` (a blinded deterministic judge). Unchanged.
- `model_corroborated` means a model panel independently supported the finding but did not deterministically establish it. A gauntlet `[V]` anchor maps here. New tier, ranked between `inference` and `verified`.
- Per-source ceilings enforce this: collectors cap at `heuristic`, `applying-formal-rigor` at `inference`, gauntlet at `model_corroborated`, and only `evidence-locked-uat` may reach `verified`. Any adapter that is handed a stronger tier than its ceiling is clamped down; a gauntlet explicit `verified` and a rigor explicit `verified` both clamp.

## Where the change lives

The record layer stays source-agnostic: `status` is still the highest tier among observations, and the new tier just sits in the ladder. Trust normalization lives at each source boundary, in the adapters, which is where knowledge of a source's strength belongs.

Design note: this makes "only a deterministic verifier reaches `verified`" a per-adapter invariant, checked by a ceiling test, rather than a global rule in `record`. A verified-source allowlist enforced in `reconcile` is a possible later hardening; the per-adapter ceilings cover it for now.

### `tests/test_tier_ceilings.py` (new file): the supported-source ceiling matrix

```
"""Supported-source tier ceilings: only a deterministic verifier reaches verified."""
from __future__ import annotations
import sys, unittest
from pathlib import Path
REPO = Path(__file__).resolve().parents[1]
sys.path.insert(0, str(REPO / "skills" / "algol" / "tools"))
import reconcile as rc  # noqa: E402
import gauntlet_adapter as ga  # noqa: E402
import compose_adapter as ca  # noqa: E402


class TestTierCeilings(unittest.TestCase):
    def test_supported_source_ceilings(self) -> None:
        # Feed each adapter input that over-claims "verified"; assert its ceiling.
        row = {"rule_id": "seclint/x", "file": "a.py", "line": 1, "standard": "security",
               "confidence": 1.0, "evidence": "[V a.py:1]", "message": "m", "col": 1}
        self.assertEqual(rc._collector_observations([row])[0][1].tier, "heuristic")

        rig = ca.parse_rigor({"findings": [{"file": "a.py", "line": 1, "claim": "c", "tier": "verified"}]})
        self.assertEqual(rig[0][1].tier, "inference")

        g = ga.parse_run_record({"verdict": "NO-GO", "findings": [
            {"file": "a.py", "line": 1, "claim": "c", "tier": "verified", "evidence": "[V a.py:1]"}]}).observations()
        self.assertEqual(g[0][1].tier, "model_corroborated")

        u = ca.parse_uat({"verdict": "FAIL", "findings": [{"file": "a.py", "line": 1, "claim": "c"}]})
        self.assertEqual(u[0][1].tier, "verified")
```
