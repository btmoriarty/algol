Algol · review control plane for Claude Code · v0.9.0

Algol, one page

The policy, evidence, and decision layer around your Claude Code reviews. Worked example captured from a real run.

Algol reviews nothing itself. It reads your versioned standards, recommends a review depth per change by policy and undo-cost, reconciles the engines' findings into one record, and keeps each human decision with the conditions that would reopen it. Sources supply the strength; Algol preserves it and never promotes a heuristic to verified on its own; the human decides.
What you get
1

Review depth by policy, not gut.

An irreversible change, like a migration or a payments path, draws a deep-review recommendation on undo-cost alone; an unmatched change like a typo does not.
2

One result record after the engines run.

Each finding is kept with its source and basis, in one record you read instead of a scrollback per tool.
3

Each finding keeps the strength its source gave it.

A scanner hit stays heuristic; verified comes only from a source that verified it, never from two guesses agreeing.
4

A disposition keeps its rationale.

Plus any conditions that would reopen it. A decision with no reopen condition is allowed but flagged.
5

The review bar is a versioned file.

Path-scoped, a contributor can read it, and changing it is a reviewable diff.

The honest gap. The in-repo tools produce four heuristic findings and no verified finding. The migration reads verified only when a real gauntlet record is folded in with a second reconcile --gauntlet gaunt.json --base .algol/record.json. That external step is not run in this harness. The model_corroborated tier is decided, not built.
What it asks of you

Write one .algol/policy.toml and evolve it through reviewed diffs. Per change: run the router, run the reviewers it recommends, reconcile their outputs, record your decision. Algol adds no new reviewer.

Where it pays off

A team on Claude Code running more than one review tool, on changes costly to reverse: security, infrastructure, permissions, data migrations. A solo weekend project will not get much from it.
